# User Prompt
When files are being downloaded, ftp server crashes

# Plan - FTP Download Crash Fix

## Problem Description
When downloading files from the FTP server, the application crashes or disconnects. This is primarily caused by:
1. **NIO Zero-Copy Crash (`FileChannel.transferTo`)**: Apache FtpServer uses `FileChannel.transferTo()` internally when sending files (the `RETR` command). On Android, `transferTo()` on SocketChannels frequently triggers native socket/kernel crashes (such as `SIGSYS` or `EXC_CRASH`) or throws unhandled IO exceptions on certain devices.
2. **Path Resolution & Security Checks**: If the FTP root directory is configured with symlinks (like `/sdcard/`), Apache FTP Server's internal path security checks may fail or raise exceptions when resolving canonical paths.

## Proposed Changes

### FTPServer Components

#### [MODIFY] [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)
- Resolve the canonical path for the user's home directory to prevent path verification security failures.
- Implement a custom `FileSystemFactory` wrapping the default `NativeFileSystemFactory`.
- Wrap the `InputStream` returned by `FtpFile.createInputStream(long offset)` inside a `BufferedInputStream`. Since `BufferedInputStream` does not inherit from `FileInputStream`, the server's internal engine will skip the `FileChannel.transferTo()` optimization path and fall back to standard buffered socket transfers, preventing native crashes.

# Plan - Fix FTP Server Crash Under Large Loads

## Problem Description
Under large loads or deep directory traversals, the application crashes. This is caused by a `StackOverflowError` due to recursive wrapping of `FtpFile` instances. In the previous fix, we used an anonymous `FtpFile` wrapper which did not prevent wrapping an already-wrapped `FtpFile`. During recursive resolution, directory listing, or repeated transfers, these wrappers nested deeply (e.g., `wrap(wrap(wrap(file)))`), eventually blowing up the call stack and crashing the application.

## Proposed Changes

### FTPServer Components

#### [MODIFY] [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)
- Create a concrete static nested class `WrappedFtpFile` implementing `FtpFile`.
- Add a helper method `wrapFile(FtpFile file)` that checks if the file is already an instance of `WrappedFtpFile`. If it is, return it directly without wrapping again.
- Update the custom `FileSystemFactory` to use `WrappedFtpFile`.

# Walkthrough - App File Logging

We have implemented file-based app logging inside the project.

## Changes Made

### Custom Logging Framework
- **Created [Log.java](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/Log.java)**:
  - Custom `Log` class that intercepts debug, info, warn, and error logs.
  - Automatically initializes a new timestamped log file (`ftpserver_yyyyMMdd.hhmmss.log`) under `/sdcard/Vypeensoft/FTP_Server/logs/` when the app starts.
  - Offloads file writing operations to a background single-threaded ExecutorService, ensuring the caller thread is never blocked.
  - Logs are duplicated to Android's built-in `android.util.Log` (Logcat) to ease debugging.

### App Lifecycle Hook
- **Created [FTPApplication.java](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPApplication.java)**:
  - Custom `Application` class that calls `Log.init()` as soon as the app process is created.
- **Modified [AndroidManifest.xml](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/AndroidManifest.xml)**:
  - Registered `.FTPApplication` to bind the new application lifecycle.

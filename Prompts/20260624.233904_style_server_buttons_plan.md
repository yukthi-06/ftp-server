# User Prompt
the text color on START SERVER and STOP SERVER is not appropriate visibly.
Also when server started, START SERVER should be disabled and also look disabled

# Initial Analysis
1. Currently, the `btnStart` and `btnStop` text colors are styled via `@color/white` in XML, but due to Material theme overrides (`colorPrimary` / `colorOnPrimary`), the text colors can inherit inappropriate colors (like blue on green/red buttons).
2. When the server starts, `btnStart` is disabled programmatically, but its background color remains green because the static `android:backgroundTint` is used in the layout. While setting its alpha to `0.5f` makes it translucent, it does not look clearly disabled as a standard Android button should (which would typically have a grey/disabled background and muted text).
3. The Stop button should also look properly disabled when the server is stopped.

# Assumptions
- The Start button must remain green (`#4CAF50`) when enabled.
- The Stop button must remain red (`#F44336`) when enabled.
- A standard grey/muted color scheme should be applied to both buttons when they are disabled.
- The text color of the buttons when enabled must be clear white (`#FFFFFF`).

# Risks
- Overriding colors programmatically might cause minor flicker or transition if not applied properly during initialization or state change, but this is negligible.

# Step-by-Step Reasoning Summary
- Define resource colors for the button states:
  - Enabled text: `#FFFFFF`
  - Disabled background (light): `#E0E0E0` (standard grey)
  - Disabled text (light): `#8E8E93`
  - Disabled background (dark): `#333333`
  - Disabled text (dark): `#777777`
- Apply these dynamically in `MainActivity.updateUI()` using `setBackgroundTintList()` and `setTextColor()`.
- Set alpha to `0.6f` when disabled to enhance the disabled visual feedback, and `1.0f` when enabled.

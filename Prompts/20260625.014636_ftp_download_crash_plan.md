# Plan - FTP Download Crash Fix

## Problem Description
When downloading files from the FTP server, the application crashes or disconnects after a short period (less than a minute). This occurs due to:
1. **System Power Management / CPU Sleep**: When the screen turns off or the app goes to the background, Android suspends the CPU and puts Wi-Fi into low-power mode to save battery. This breaks active socket connections.
2. **Missing WakeLock and WifiLock**: The background service does not hold a `PowerManager.WakeLock` and `WifiManager.WifiLock`, leaving it vulnerable to being suspended/killed by the OS during active transfers.
3. **Foreground Service Type Specification**: Under Android 10 (API 29) and later, when launching a foreground service with a declared type (`dataSync` in our manifest), we should specify that type in `startForeground()`.
4. **Network on Main Thread**: Starting and stopping the FTP server involves socket binding/unbinding operations which run synchronously on the calling thread. If done on the main thread, this can lead to responsiveness issues or strict mode crashes.

#### [MODIFY] [FTPServerService.java](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerService.java)

- Add member variables for `PowerManager.WakeLock mWakeLock` and `WifiManager.WifiLock mWifiLock`.
- In `startFtpServer()`, run the server start logic on a background thread.
- Acquire `WakeLock` and `WifiLock` when the server starts to prevent the CPU and Wi-Fi from going to sleep.
- Update `startForeground()` call: if API level >= 29, specify `ServiceInfo.FOREGROUND_SERVICE_TYPE_DATA_SYNC`.
- In `stopFtpServer()` and `onDestroy()`, release the locks and stop the server on a background thread.

# Task Journal - Fix FTP Server Upload Transfer Crash Under Large Loads

## Initial Analysis
While download crashes were resolved by wrapping the input stream, upload operations (writing files onto the server) still use native `FileChannel.transferFrom()` which triggers SIGABRT crashes under heavy loads on Android.

## Design Decisions
- Wrap the output stream returned by `createOutputStream` in a `BufferedOutputStream` to prevent zero-copy write optimizations.

## Implementation Plan
1. Update `WrappedFtpFile.createOutputStream` to wrap the stream in a `BufferedOutputStream`.

# User Prompt
Create a complete Android application in **Java (NOT Kotlin)** named **FTP Server**.

## Objective

Build a lightweight FTP Server Android app that allows the user to start and stop an FTP server on their device and manage all configuration through a JSON settings file.

## Technical Requirements

### Platform

* Android Studio project
* Language: Java only
* Minimum SDK: Android 8.0 (API 26)
* Target latest Android SDK
* Material Design UI
* No Kotlin files
* Gradle build should compile successfully

### FTP Server Features

Implement a fully functional FTP server running locally on the Android device.

Supported features:

* Start FTP Server
* Stop FTP Server
* Display server status (Running / Stopped)
* Display FTP URL
* Display device IP address
* Display listening port
* Show connected clients count
* Anonymous login option
* Username/password authentication option
* Passive mode support
* Read-only mode option
* Allow upload/download/delete operations when permitted
* Auto-start server on boot (optional setting)
* Foreground service while server is running
* Persistent notification showing server status

Use a reliable Java FTP server library if needed.

# 🛠️ Implementation Plan: Android FTP Server Application (Java)

## Initial Analysis & Approach

We need to build a lightweight, Material Design FTP Server application using Java in an Android Studio project structure. The app must run an FTP server in a foreground service, load and save config from a custom path (`/sdcard/Vypeensoft/FTP_Server/setings/settings.json`), handle Android permissions (including storage access), support auto-boot, and provide UI to control the server and modify settings.

### Key Components

1. **Gradle Restructuring**:
   - Clean up Sokoban references in `settings.gradle` and define a single-module `:app`.
   - Add the necessary dependencies: `org.apache.ftpserver:ftpserver-core`, SLF4J Android logging binding, Gson, and Material Design UI.
2. **Settings Storage & Manager (`SettingsManager.java`)**:
   - File path: `/sdcard/Vypeensoft/FTP_Server/setings/settings.json` (also fallback to `Environment.getExternalStorageDirectory()` for portability).
   - Use Gson for reading/writing settings. Handle file generation, directory verification, and fallback recovery.
3. **FTP Server Implementation (`FTPServerManager.java`)**:
   - Wrap Apache FtpServer.
   - Configure port, passive mode, anonymous log-in, read-only permissions, and maximum connections.
   - Track active connections.
4. **Foreground Service (`FTPServerService.java`)**:
   - Runs the FTP server in a foreground service.
   - Posts a persistent notification with the server IP/URL and start/stop controls.
5. **Network Utilities (`NetworkUtils.java`)**:
   - Get active Wi-Fi IPv4 address.
   - Build FTP URLs.
   - Validate port status.
6. **UI Screens**:
   - `MainActivity.java`: Control panel showing server status, IP, port, FTP URL, client count, and start/stop buttons.
   - `SettingsActivity.java`: Edit properties (port, user credentials, root folder, permissions, etc.) with Save & Reset buttons.
7. **Boot Support (`BootReceiver.java`)**:
   - Listens to `ACTION_BOOT_COMPLETED`.
   - Checks `auto_start` setting; if true, starts the service.

---

## User Review Required

> [!IMPORTANT]
> **Storage Access on Android 11+ (API 30+)**:
> Since the target location is `/sdcard/Vypeensoft/FTP_Server/setings/settings.json`, which lies outside standard app-private storage, the app must request `MANAGE_EXTERNAL_STORAGE` (All Files Access) on Android 11+ and legacy write permissions on older versions. We will implement code to request this permission gracefully in UI.

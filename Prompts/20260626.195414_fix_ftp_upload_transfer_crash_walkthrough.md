# Walkthrough - Fix FTP Server Upload Transfer Crash Under Large Loads

We have successfully resolved the FTP server upload crashes under large loads. Below is a summary of the changes implemented.

## Changes Made

### Files Modified
- [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)

### Key Improvements
1. **Prevented Native transferFrom Crashes on Uploads**:
   - Wrapped the returned `OutputStream` from `createOutputStream(long offset)` inside a standard `java.io.BufferedOutputStream` in `WrappedFtpFile`.
   - This ensures that when the FTP client uploads files (the `STOR` command), the server's data transfer engine (`IODataConnection`) does not recognize it as a raw `FileOutputStream` and bypasses the native `FileChannel.transferFrom()` zero-copy method (which can trigger native `SIGABRT` crashes on Android under heavy/high-throughput loads).

# Task Journal - Fix FTP Server Crash Under Large Loads

## Initial Analysis
Under large loads, the FTP server crashes. The root cause is a `StackOverflowError` triggered by the recursive nesting of anonymous `FtpFile` wrappers when they are repeatedly resolved or listed.

## Design Decisions
- Convert the anonymous `FtpFile` wrapper to a concrete `WrappedFtpFile` static nested class.
- Add an `instanceof` check in the helper `wrapFile()` to avoid wrapping already wrapped files.

## Implementation Plan
1. Define class `WrappedFtpFile` in `FTPServerManager.java`.
2. Implement `wrapFile()` to filter out already wrapped instances.
3. Update `FileSystemFactory` to use `wrapFile()`.

### Decisions Made During Implementation
- Replaced the anonymous inner class FtpFile implementation with a static nested class `WrappedFtpFile` to handle structural wrapping.
- Implemented the static `wrapFile(FtpFile file)` helper to verify if the file is already a `WrappedFtpFile` instance, returning it directly if true. This breaks the infinite wrapping loop during repeated traversals and file operations under large loads.

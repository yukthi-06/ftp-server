# Plan - Fix FTP Server Upload Transfer Crash Under Large Loads

## Problem Description
The application continues to crash under large loads. The crash dump shows native `SIGABRT` crashes on thread pools during transfers. Since we resolved download operations by wrapping the `InputStream` in a `BufferedInputStream` (disabling `FileChannel.transferTo`), the remaining crashes during file upload/store operations are caused by the native `FileChannel.transferFrom()` optimization on the `OutputStream` (since `createOutputStream` was returning a raw `FileOutputStream`).

## Proposed Changes

### FTPServer Components

#### [MODIFY] [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)
- In the static nested class `WrappedFtpFile`, wrap the output stream returned by `createOutputStream(long offset)` inside a standard `java.io.BufferedOutputStream`.
- This ensures that the type check `outputStream instanceof FileOutputStream` fails inside the server's transfer engine, forcing it to fall back to safe, standard buffered stream-based socket writes for uploads as well.

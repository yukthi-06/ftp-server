# User Prompt
When files are being downloaded, ftp server crashes

# Task Journal - FTP Download Crash Fix

## Initial Analysis
The FTP server successfully starts and accepts client control connections. However, during file retrieval (download), it crashes. This occurs due to:
1. **NIO transferTo zero-copy socket transfer issues on Android**: Android devices/kernels often crash natively (such as SIGSYS) or throw IOException when executing `FileChannel.transferTo` on a socket channel.
2. **Canonical vs Absolute symlink verification issues**: Direct use of paths like `/sdcard/` might fail security boundaries checked by Apache FtpServer when resolving relative paths against the user's home directory.

## Design Decisions
- Wrap `FtpFile.createInputStream` to return `BufferedInputStream` instead of raw `FileInputStream`. This prevents `IODataConnection` from executing `FileChannel.transferTo` and makes it fall back to standard stream copy.
- Resolve canonical paths for the user's home directory.

## Implementation Plan
1. Modify `FTPServerManager.java` to canonicalize the home directory.
2. Wrap the `NativeFileSystemFactory` in a custom `FileSystemFactory` that wraps all returned `FtpFile` instances and wraps their input stream in a `BufferedInputStream`.

### Decisions Made During Implementation
- Replaced the direct use of standard `NativeFileSystemFactory` with an anonymous subclass of `FileSystemFactory` that intercepts `createFileSystemView` to return a wrapped `FileSystemView` and `FtpFile`.
- Inside `createInputStream` of the wrapped `FtpFile`, the returned raw `InputStream` is wrapped inside `java.io.BufferedInputStream`. This changes its runtime type checking so it is no longer recognized as a `FileInputStream`, causing the Apache FTP Server's data connection handler to fallback to standard buffer transfer loops instead of native `FileChannel.transferTo()`.
- Updated `CustomUserManager`'s home directory getter to resolve standard canonical paths via `getCanonicalPath()` to prevent directory boundary checks from failing due to symlink mismatch (e.g. `/sdcard` vs `/storage/emulated/0`).

# Plan - Optimize Settings IO and Cache Network Queries

## Problem Description
The app still crashes/hangs under load (e.g. during active file transfers). This is highly likely due to:
1. **Disk I/O on the Main Thread**: `MainActivity.updateUI()` calls `SettingsManager.loadSettings()` every second. Under high I/O load (during file transfer), reading the settings JSON from disk blocks the main thread, resulting in an ANR (Application Not Responding) crash.
2. **Frequent Network Queries on the Main Thread**: `NetworkUtils.getWifiIpAddress()` is called every second in `updateUI()`. Fetching Wi-Fi info and listing network interfaces under heavy network transfer can block the main thread.
3. **Synchronous File Writes on Main Thread**: `SettingsManager.saveSettings()` performs blocking file writes on whichever thread calls it, including the main thread.

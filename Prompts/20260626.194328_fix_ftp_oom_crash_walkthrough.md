# Walkthrough - Fix FTP Server Crash Under Large Loads

We have successfully resolved the FTP server crash encountered under large loads. Below is a summary of the changes implemented.

## Changes Made

### Files Modified
- [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)

### Key Improvements
1. **Prevented StackOverflowError from FtpFile Wrappers**:
   - Converted the anonymous `FtpFile` wrapper to a concrete static nested class `WrappedFtpFile`.
   - Added a static `wrapFile(FtpFile file)` helper method that checks if the file is already an instance of `WrappedFtpFile` before wrapping. If it is, the file is returned as-is.
   - This prevents deep wrapper chain nesting (e.g. `wrap(wrap(wrap(file)))`) that previously exhausted the JVM call stack during repeated file resolution and directory list operations under large loads.

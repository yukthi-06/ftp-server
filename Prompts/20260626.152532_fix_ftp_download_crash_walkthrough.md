# Walkthrough - FTP Download Crash Fix

We have successfully resolved the crash/disconnect issues encountered during FTP file downloads. Below is a summary of the changes implemented.

## Changes Made

### Files Modified
- [FTPServerManager.java](file:///x:/Projects_X/0_Active/8_Android_APK/02_Stable/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerManager.java)

### Key Improvements
1. **Prevent NIO transferTo zero-copy crashes on Android**:
   - Implemented a custom `FileSystemFactory` that wraps all returned `FtpFile` files.
   - Wrapped the returned `InputStream` from `createInputStream(long offset)` inside a standard `BufferedInputStream`.
   - This hides the `FileInputStream` type from Apache FtpServer's internal engine (`IODataConnection`), safely forcing it to bypass the native `FileChannel.transferTo()` optimization path (which triggers native socket crashes on Android) and fall back to standard buffered socket transfers.
2. **Canonicalize Home Directories**:
   - Added canonical path resolution (`getCanonicalPath()`) to `CustomUserManager` for both normal and anonymous users. This ensures that symlinked root paths like `/sdcard` are properly evaluated against physical/canonical path representations (`/storage/emulated/0`), preventing internal path validation check failures.

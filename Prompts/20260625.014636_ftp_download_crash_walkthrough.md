# Walkthrough - FTP Download Crash Fix

We have successfully resolved the crash/disconnect issues encountered during FTP file downloads. Below is a summary of the changes implemented.

## Changes Made

### Android Manifest
- Added `<uses-permission android:name="android.permission.WAKE_LOCK" />` to [AndroidManifest.xml](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/AndroidManifest.xml) to authorize acquiring WakeLocks.

### Foreground Service
- Modified [FTPServerService.java](file:///x:/Projects_X/0_Active/8_Android_APK/05_Early_Stage/FTP_Server_APK_GITHUB/app/src/main/java/com/vypeensoft/ftpserver/FTPServerService.java):
  - **WakeLock & WifiLock Implementation**: Added support for acquiring a `PARTIAL_WAKE_LOCK` (CPU) and a `WIFI_MODE_FULL_HIGH_PERF` or `WIFI_MODE_FULL` WifiLock when starting the FTP server. These locks are cleanly released when stopping or destroying the service.
  - **Background Thread Execution**: Shifted blocking network operations (socket checking/binding, server startup and shutdown) to a background thread to prevent `NetworkOnMainThreadException` or thread hanging on the UI main thread.
  - **Foreground Service Type Specification**: Updated the `startForeground()` call to pass the `ServiceInfo.FOREGROUND_SERVICE_TYPE_DATA_SYNC` type when running on Android 10 (API 29) or later, matching its manifest declaration.
  - **Volatile State Variable**: Marked `mIsRunning` as volatile to ensure proper thread safety and visibility across the main thread and background server thread.
